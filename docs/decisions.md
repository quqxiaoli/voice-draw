# Decisions · 架构决策记录

> 格式:决策 / 原因 / 备选(为什么没选)。按时间序。

## D1 · 选题
- **决策**:选题目二(AI 语音绘图工具)
- **原因**:ASR 为两题共同成本可抵消,选题只比下游——题二下游为 LLM 结构化解析 + 本地 Canvas 确定性执行(零图像 API、可复录),题一下游为 VLM+TTS 实时循环(3 模型依赖、"自然度"受端到端延迟支配不可控);"复杂指令拆解"同时命中评分项、demo 亮点、设计文档主章节
- **备选**:题目一,未选因实时音视频交互压在前端零基础短板上,且自然度指标在云 API 链路下 72h 不可工程化保底

## D2 · 绘图引擎与呈现
- **决策**:绘图引擎定为 SVG 矢量画布 + 有限图形原语,呈现采用"SSE 流式逐命令执行 + 描边动画"
- **原因**:矢量命令满足确定性/可撤销/可逐步执行,描边动画将延迟转化为表演性,零新增依赖
- **备选**:raster 或文生图(随机性不可演示、不可撤销)、流程图专用模式(指令空间发散偏题),均砍

## D3 · 架构表述(乘法叙事)
- **决策**:架构表述为"语音→ASR→LLM编译→绘图DSL→SVG渲染"四段编译器结构,DSL 为核心抽象层
- **原因**:DSL 隔离输入输出两端,支撑"架构乘法"叙事(多输入多渲染端可替换)与商业叙事(无障碍创作);表达力上限由 LLM path 生成质量决定,开赛实测后定指令集边界,架构不随实测结果变
- **备选**:无 DSL 直出 SVG 字符串(LLM 直接写 SVG),未选因不可撤销、不可逐条流式执行、容错无从下手

## D4 · ASR 方案
- **决策**:ASR 采用浏览器 Web Speech API,云 ASR(后端转发)为降级备胎,开赛 2h 内 10 句指令实测定案
- **原因**:零成本(设计文档成本控制第一条)、零后端、自带流式中间结果(容错 UI 的基础)、前端工程量最小;备胎仅在识别不可用时启用
- **备选**:云 ASR 主选,未选因增加录音上传链路(压前端短板)、增加延迟与成本,且其收益(识别率)可被容错机制部分兜底

## D5 · LLM 供应商
- **决策**:LLM 走 OpenAI 兼容协议,base_url/key 环境变量化,七牛云推理为主选、DeepSeek 官方为热备,开赛实测定主选
- **原因**:评委亲和叙事 + 协议兼容使切换成本为两行配置
- **备选**:硬编码单一供应商,未选因上游故障无逃生口

## D6 · 歧义处理(不做生成前反问)
- **决策**:不做生成前澄清反问,歧义处理 = 修正指令迭代(改色/改大小/移动/撤销)+ 无法解析时单条 clarify 提示
- **原因**:反问阻塞"语音→即画"主链路即时性,修正式迭代同时是"复杂指令拆解"评分项的展示场;绘画工作流天然迭代而非问询
- **备选**:对话式多轮澄清,未选因增加交互轮次、demo 节奏被打断、且需求已被撤销+修正覆盖

## D7 · 零数据库
- **决策**:零数据库(无 SQLite);新增前端导出 PNG/SVG + 会话内命令历史侧栏
- **原因**:单画布单会话,状态在前端,后端无持久化需求;演示场景的"留存"需求 = 成果带走(导出,纯前端)+ 过程回看(命令栈复用渲染),均零后端成本;跨会话持久化属运营期需求;每删一个组件就少一类故障
- **备选**:SQLite 存历史作品,未选因引入存储+列表页+加载逻辑约 1.5 个 PR 工作量,服务于评审场景中不存在的用户行为

## D8 · 后端唯一状态:会话上下文
- **决策**:后端唯一状态 = 会话对话上下文(修正指令的指代消解),SessionStore 接口 + 内存 map 实现(最近 N=20 条命令),不落盘
- **原因**:真需求(LLM 需要最近命令上下文)驱动;接口化保留替换 Redis/SQLite 的插座,service 零改动——以可扩展性设计而非组件堆叠展示工程能力
- **备选**:无状态后端每次全量传上下文(前端带历史),未选因请求体膨胀且上下文管理逻辑泄漏到前端;直接上 Redis,未选因单实例内存已覆盖需求

## D9 · SSE token 事件语义修订(对 constitution 第 10 节的项目级修订)
- **决策**:SSE token 事件 content 修订为完整 DrawCommand 对象(每事件一条命令),后端攒齐再发,不透传原始 token
- **原因**:前端逐命令执行+描边动画要求事件原子性,半条 JSON 不可执行;三事件结构(token/done/error)与前端骨架不变
- **备选**:透传文本增量前端自行切分 JSON,未选因解析逻辑泄漏到前端且断流时半条命令难处理
- **修订(T+15h)**:token 事件 data 为 DrawCommand 直读,取消 content 包装;op 定名 `draw`(弃 `add`),结构扁平化——以骨架实现为准回写契约

- 决策:三项 go/no-go 取消,改为假设+兜底:① LLM 主选切 DeepSeek 官方(规则确认无强制七牛),七牛降为备选,Gemini 否决(协议差异需单独 client 路径);② 指令集采用混合策略:基础原语 + path 一等原语,prompt 鼓励有机形体用 path、几何物体用原语组合,激进程度开发期可调;③ 语音(Web Speech)为主输入不变,新增并列文本输入框作降级通道,共用下游链路
