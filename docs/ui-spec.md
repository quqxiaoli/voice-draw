# UI Spec · 页面与交互

> UI 先行。先定 UI,后端按契约服务这个 UI,不旁逸斜出。
> 设计基调已赛前定稿(下方 token),议题来了只填页面部分。

## 设计基调(已定稿,议题无关)

- **风格**:日系简洁 —— 大量留白、柔和低饱和配色、克制装饰、内容优先。在一堆"能用就行"的作品里靠质感突出。
- **设计 token**(Tailwind 配置 / CSS 变量集中定义,所有组件从 token 取值,不写死任意值):

| token | 值 | 用途 |
|---|---|---|
| `--bg` | `#FAF9F6` | 页面底色(暖白,不用纯白) |
| `--surface` | `#FFFFFF` | 卡片/输入区表面 |
| `--text` | `#3D3D3D` | 正文(暖深灰,不用纯黑) |
| `--text-muted` | `#8A8A85` | 次要文字/占位符 |
| `--accent` | `#7C9A83` | 主题色(灰调松绿,Yuru Camp 方向) |
| `--accent-soft` | `#EAF0EB` | 主题色浅底(hover/选中背景) |
| `--border` | `#E8E6E1` | 分割线/描边(极浅,弱化边界感) |
| `--danger` | `#C77B6E` | 错误(柔和砖红,不用刺眼正红) |

- **字体**:中文 `"Noto Sans SC"`,西文/数字 `Inter`,等宽 `"JetBrains Mono"`(代码/ID 展示用)。正文 15px/1.7 行高,标题层级最多 3 级。
- **圆角**:统一 `10px`(卡片、按钮、输入框同一个值,克制,不混用多种圆角)。
- **间距基准**:8px 网格(8/16/24/32/48),区块之间宁可空一点,不挤。
- **阴影**:几乎不用。最多 `0 1px 3px rgba(0,0,0,0.04)` 一档,靠留白和浅边框分层,不靠阴影。
- **动效**:只做两种 —— 状态过渡 `150ms ease` + 流式打字机。不加多余入场动画。

## 页面规格:以静态稿为准

> 决策(赛前定):页面布局/交互不写文档,**v0 定稿落库后的静态页面本身就是规格**,
> 视觉判断靠人眼在浏览器里做。本文件只管两件事:① 上面的设计 token ② 下面的状态底线。
> DS 新增任何 UI 元素(按钮/提示条/弹层)时,颜色间距圆角必须从 token 取——
> 这是人眼两次验收之间防止风格漂移的唯一机制,不可删。

## 状态底线(每个页面必须做出、可切换预览)

> v0 的稿子通常只有 happy path,以下四态由 DS 落库时补齐,人眼逐个验收:

- **空态**:一句引导文案 + 示例入口,不留白板
- **loading/流式中**:打字机逐字渲染;操作按钮禁用;可中断则给停止按钮
- **成功态**:结果可复制/导出;给下一步动作
- **错误态**:柔和提示条(--danger 浅底),保留用户输入,给重试按钮,绝不白屏
