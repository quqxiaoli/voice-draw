# Spec · 作品规格

## 议题
题目二:AI 语音绘图工具
请开发一款纯语音控制的绘图工具。
要求:用户不能使用鼠标或键盘,仅通过语音指令完成绘图创作。请综合考虑指令理解的准确性与容错性、语音到绘图操作的响应延迟,以及复杂指令的拆解与执行能力。实现应用的同时,请额外提交一份设计文档,记录你计划支持哪些指令能力,最终实现了哪些,以及未完成部分的原因说明。

## 一句话定义
动嘴作画:把语音指令实时编译成结构化绘图命令,在 SVG 画布上一笔一笔画出来。

## 目标用户 & 使用场景
双手不便使用鼠标键盘的创作场景:无障碍创作(运动障碍人士)、不会打字的儿童涂鸦启蒙、手忙场景下的快速示意(厨房/车载/白板讲解)。比赛约束"不能用鼠标键盘"在这些人群里是真实刚需,不是人为限制——本作品把约束升格为产品定位。
核心痛点:现有绘图工具的交互全部建立在手部精细操作上,语音用户的创作表达为零。本作品提供的不是"语音版 Photoshop",而是最低门槛的"说出来就画出来"。

## 核心闭环(唯一,必须完整)
1. 用户点击麦克风开始说话(开关式,再点一次结束并提交)→ 浏览器 Web Speech API 实时转文字,识别中间结果回显在界面上;并行保留文本输入框作降级与键盘通道
2. 文字指令 POST 到 Go 后端 → LLM 编译为绘图 DSL 命令(JSON),SSE 流式逐条返回
3. 前端每收到一条命令即执行:图形以 600ms 描边动画"画"出来(克制单一动效,实测 300–500 偏短);支持修正指令(改色/改大小/移动/撤销)对画布迭代

## 做减法:明确砍掉什么
- 不做:raster/油画/文生图风格　理由:随机性不可演示不可撤销;语音的低信息带宽驱动不了笔触级表达,输入输出带宽应匹配
- 不做:生成前澄清式反问　理由:阻塞"说话→即画"即时性;歧义由修正指令迭代解决,符合绘画天然工作流
- 不做:流程图/图表专用模式　理由:指令空间发散偏题,基础图形可拼即够
- 不做:多画布/作品管理/用户体系/跨会话持久化　理由:单画布单会话即闭环;留存需求由导出(成果带走)+ 命令历史侧栏(过程回看)覆盖,跨会话管理属运营期需求
- 不做:语音播报回应(TTS)　理由:视觉反馈(画布+文字回显)已闭环,TTS 加延迟加依赖不加分
- 不做:移动端适配　理由:演示在桌面 Chrome,README 声明环境

## 指令集 V1(≤13 条,设计文档"计划支持"章节同源)

**分支 A(开赛实测 LLM path 质量 ≥ 可用):**
1. 画基础图形:圆/矩形/线/三角(含颜色、大小、位置参数)
2. 自由图形:「画一只猫/一棵树」→ LLM 生成 path
3. 复合场景:「画一个雪人」→ 拆解为多条原语命令
4. 修改颜色　5. 改大小　6. 移动　7. 删除指定图形　8. 撤销上一步
9. 清空画布　10. 写文字　11. 填充/描边切换　12. clarify 兜底提示　13. 导出(语音触发;按钮并存,导出发生在创作完成后不违题)

**分支 B(path 质量差):砍掉第 2 条,自由图形降级为预制模板库(猫/树/房子/太阳等 8 个),其余不变。**
复合拆解(第 3 条)两分支都保留——它是评分项核心,分支 B 下拆解目标为原语+模板。

## 完整度清单(核心闭环要满足)
- [ ] 识别中文字回显 + LLM 编译中状态 + 描边动画(流式反馈三连)
- [ ] 错误态:ASR 不可用提示换 Chrome;LLM 失败/超时 SSE error 事件,画布状态不丢,可重说
- [ ] 空态:首屏画布带语音提示引导("试试说:画一个红色的圆")
- [ ] 边界:超长语音、静音、无法解析指令(clarify)、连续快速指令不崩
- [ ] 主链路从头到尾可演示:说雪人→拆解流出→逐笔画出→"帽子改成红色"→修正生效→"撤销"

## 创新点 / 差异化
流式逐笔作画:LLM 命令流式返回 + 描边动画,延迟被转化为"AI 执笔表演"——不加功能,只改呈现节奏。辅以编译器架构叙事(语音→ASR→LLM编译→DSL→渲染,DSL 为乘法抽象层)。

## 技术栈决策(constitution ⚠️ 项回填,详见 decisions.md)
- 数据库:**零数据库**——单会话无持久化需求,画布状态在前端内存;后端唯一状态为会话上下文(SessionStore 接口 + 内存 map)
- Redis:不上,无用户可感知收益;SessionStore 接口化已留替换插座
- 流式:浏览器直连 Go 后端 SSE(默认)
- ASR:浏览器 Web Speech API,零后端零成本;云 ASR 备胎(仅识别不可用时启用)
- LLM:OpenAI 兼容协议,七牛云推理主选 / DeepSeek 热备,.env 切换
